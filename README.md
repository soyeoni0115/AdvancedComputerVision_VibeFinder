# AdvancedComputerVision_VibeFinder

## 해당 프로젝트는 고급컴퓨터비전 기말고사 대체 프로젝트로 진행되었음

## 3. 데이터 확보 및 처리 계획
- 데이터셋 출처: (예: AI Hub, Kaggle, Roboflow Universe, 직접 수집 등) SNS 크롤링 데이터, AI Hub(한국형 관광 명소 데이터), Roboflow Universe 등
	- 데이터 규모: (예: 클래스 개수, 총 이미지 수, 비디오 분량 등) 최소 1000장의 감성 태그가 포함된 여행지 사진 쌍(Pair) 구성. 
	- 전처리 전략: (Augmentation 기법, 이미지 리사이징 전략 등) CLIP 모델이 구도적 특징을 잘 학습하도록 이미지 리사이징 및 Center Crop/Random Flip 등 적용
- 조도 및 색감을 학습하도록 Color Jittering(밝기, 대비 조절)/Gaussian Blur 등 적용
	- 촬영 시간대와 날씨에 따라 감성이 크게 달라질 수 있으므로 학습이 필요함
	- LLM 기반 paraphrasing: “따뜻한 느낌의 카페”라는 단일 라벨을 gpt 등 llm을 활용해 “채광이 잘 드는 카페” 등으로 확장하여 텍스트 임베딩의 범용성을 높일 수 있음
	- Augmentation을 활용하여 데이터셋 부족을 해결 예정
- 라벨링 계획: (라벨링 툴 선정 및 작업 분담 계획) Roboflow를 사용해 이미지마다 "따뜻한", "아기자기한", "미니멀한" 등 감성 레이블링 수행 
	- 감성 태그 체계 구축: 단순한 형용사 나열이 아닌, 시각적 요소와 감정을 결합한 계층적 라벨링을 수행함
	    - 분위기: 빈티지한, 미니멀한, 따뜻한, 힙한, 고즈넉한 등
	    - 시각적 요소: 자연광, 네온사인, 원목 위주, 파스텔톤, 대칭 구도 등
	    - 장소 속성: 도심 골목, 바다 근처, 실내 카페, 야경 스팟 등
	- 라벨링 품질 확보: 교차 검증, 불일치 해결
	    - 동일 이미지에 대해 팀원 각자 라벨링을 수행하고, 2인 이상 일치하는 태그만 최종 데이터셋으로 채택함. 합의가 이루어지지 않은 이미지는 미정 클래스로 분류하거나 삭제하여 데이터셋의 순도를 유지함.

 
## 4. 기술 스택 및 모델 구조
### 4.1 사용 모델 및 프레임워크
- Detection/Tracking/Embedding: CLIP-ViT-B/32 ( LoRA 기반 경량 파인튜닝 필요)
 
- Backend/Inference: PyTorch, ONNX Runtime

- DB: Vector DB - FAISS (데이터가 늘어날 것을 대비해 IndexIVFFlat 와 같은 인덱싱 기법도 고려)
 
- Frontend/UI: Streamlit
 
### 4.2 시스템 아키텍처
사용자 텍스트 쿼리 → CLIP Text Encoder → 벡터 변환 → FAISS Vector DB 내 이미지 벡터와 코사인 유사도 비교 → 가장 유사도가 높은 여행지 이미지 및 관련 태그 UI 출력

- 김소연

	- 데이터 전처리 및 증강
	- 이미지 캡션 생성
	- CLIP 모델 불러오기 및 전이학습
	- Streamlit 화면 구성
- 이연우
	- 데이터를 수집함으로써 lora 학습의 밑바탕을 제공

	- faiss db 구축으로 자연어 입력 시 알맞는 이미지를 잘 조회할 수 있도록 함
	- 네이버 api연결하여 데이터 크롤링

## 상세 설치/실행방법
### 실행 방법
- 1. streamlit run app.py로 실행
- 2. 자연어로 원하는 카페를 입력 혹은 태그를 선택한 후 검색하기 버튼을 클릭
- 3. lora가 불려지면 왼쪽엔 카페이미지, 오른쪽엔 카페 이름과 카페의 주소, 카페 길찾기가 존재한다.

## 데이터 파이프라인

데이터 -> 전처리 -> 증강 -> 일반 데이터 faiss db에 넣기
	  -> PostgreSQL(neon)에 이미지 데이터 경로, 이미지(카페) 이름, 데이터 파일명, 카페 주소, 카페 길찾기 연결 등 streamlit과 lora 학습에 필요한 정보들을 저장 
	=> PostgreSQL에서 캡션을 가져와 raw이미지와 증강 이미지를 사용해 lora 학습
	=> lora가 켜지면 faiss db에서 이미지 데이터를 가져오고 이미지에 맞는 정보를 불러옴
